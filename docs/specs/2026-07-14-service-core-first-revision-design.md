# 服務核心第一階段修正設計

- 日期：2026-07-14
- 狀態：已核准（2026-07-14）
- 設計決策：`DEC-022`～`DEC-026`
- 數值依據：`DB-003`
- 第二階段待決：`OPEN-010`

## 1. 目的

本設計修正首版圖集對基地朝向、L1 男女廁、L2 更衣淋浴樓層外框、樓梯中央落點及玻璃屋頂範圍的錯誤。第一階段先建立可立即修訂的概念模型；`RF-GL-01` 與二樓擴建量體的施工式交界構造留待第二階段討論。

## 2. 已確認的空間關係

1. `CORE-01` 必須落在衛星圖綠色原廁所基地，`EN-01` 必須對應紅色箭頭，泳池大廳向黃色基地延伸。
2. 本地 +X 由泳池遠端指向服務核心，工作方位為 307°；不得只旋轉 `REF-001`，所有圖面與後續 Viewer 必須共用同一 transform。
3. L1 服務核心維持原基地 11.0 × 13.5 m，男女廁採男左女右。
4. L2 不受 L1 外框限制：除原服務核心上方外，再向泳池大廳上方伸出二樓專用擴建量體；其下方 L1 不新增封閉房間。
5. L2 採男左女右，`ST-01` 上端位於兩區中央；男女入口錯位且有視線遮擋。
6. `RF-GL-01` 從二樓擴建量體邊緣向泳池遠端左下傾斜 10°。

## 3. 可修改的首版參數

首版將 `building.l2ExtensionLength = 5.0 m` 設為 `working`。這不是不可更動的建築尺寸；未來只需修改此一基礎參數，所有衍生幾何必須重新計算。

| 衍生項目 | 公式 | 首版結果 |
| --- | --- | --- |
| L2 起點 X | `poolHallLength - l2ExtensionLength` | 19.0 m（D 軸） |
| L2 工作長度 | `serviceCoreLength + l2ExtensionLength` | 16.0 m |
| L2 中央分流軸 | `(l2StartX + building.length) / 2` | 27.0 m（E 軸） |
| 男生區 X 範圍 | `l2StartX .. l2SplitAxisX` | 19.0～27.0 m |
| 女生區 X 範圍 | `l2SplitAxisX .. building.length` | 27.0～35.0 m |
| 屋頂平面跨度 | `poolHallLength - l2ExtensionLength` | 19.0 m |
| 樓梯起點 X | `l2SplitAxisX - stairTotalRun` | 19.04 m（以現有 7.96 m 總水平長度推導） |

renderer 不得再各自寫入 `19`、`16`、`27` 或 `19.04`。驗證器應修改 `l2ExtensionLength` 後重新推導並檢查六張圖仍一致。

## 4. L1 男女廁與入口

- 男廁 `Z-WC-M-01` 位於較低 X 側，維持 3 座坐式馬桶、3 座小便斗及洗手台。
- 女廁 `Z-WC-F-01` 位於較高 X 側，維持 5 座坐式馬桶及洗手台。
- 新增 `Z-L1-ENTRY-01` 表達操場側有遮蔽的方形共用前室。原稿只確認退縮關係，精確尺寸維持 `deferred`，首版不得顯示未經確認的數字。
- `EN-01` 由操場進入共用前室；男女廁各自以前門連接前室。
- 男女廁各自以後門連接泳池後側乾式通道。兩樘後門獨立管制，只在泳池使用時段開啟。
- 同一廁所的前、後門不得正對；門口以錯位及短遮擋牆避免從操場側看穿到泳池側。
- `ST-01` 與廁所動線獨立，不得阻斷共用前室及四樘廁所門。

## 5. L2 更衣淋浴與擴建量體

- 新增 `EXT-L2-01` 表達 X=19.0～24.0 m 的二樓專用擴建量體；它與原核心上方合併成 X=19.0～35.0 m 的 L2 工作外框。
- `EXT-L2-01` 下方為 L1 開放池畔／泳池大廳，不建立對應的 L1 房間。
- `Z-CS-M-01` 與 `Z-CS-F-01` 在 E 軸分開，兩區面積相等，各維持 15 間正式整合單元及 5 間擴充位置。
- `ST-01` 二樓上端落在 E 軸中央分流前室；男生向較低 X 側，女生向較高 X 側。
- 擴建量體與玻璃屋頂結構獨立。淋浴、排水、防水及結構配置屬 `OPEN-009` 的後續專業設計，不在概念圖集中宣稱施工可行。

## 6. 第一階段屋頂表達

- `RF-GL-01` 的 10° 坡度與向泳池遠端左下傾斜的方向已確認。
- 屋頂平面高端終止於 `EXT-L2-01` 的低 X 邊緣；5.0 m 擴建值改變時，屋頂平面跨度同步改變。
- 既有 +6.000／+10.231 m 是以錯誤的 24.0 m 跨距推導，不得繼續標示為當前答案。
- 第一階段剖面以 `deferred` 接點表達屋頂與擴建量體的關係，不標註最終高低標高，也不畫施工式密封細節。
- 新增 `J-RF-L2-01` 作為第二階段交界設計的穩定 ID；端面材質、密封、活動縫、泛水、排水、冷凝、防潮與斷熱均由 `OPEN-010` 管理。

## 8. 驗收條件

1. 所有朝向只讀單一 307° transform；`CORE-01`、`EN-01` 與泳池延伸方向符合衛星標註。
2. 修改 `l2ExtensionLength` 時，L2 起點、總長、男女分流軸、樓梯上端及屋頂跨度一起更新。
3. L1 男女廁左右排列，前後門均存在、相互錯位，泳池側後門標示為時段管制。
4. L2 男女區面積相等、嚴格分離，各有 15＋5 單元，樓梯上端位於兩區中央。
5. `EXT-L2-01` 下方沒有 L1 房間，且其結構不依賴 `RF-GL-01`。
6. `J-RF-L2-01`、屋頂最終標高及施工式交界在第一階段保持 `deferred`。
7. validator、Node tests、TypeScript build、桌面／手機 browser smoke 與 `git diff --check` 全部通過後，第一階段才算實作完成。

## 9. 第二階段邊界

第二階段只在使用者另行核准後開始，處理 `J-RF-L2-01` 的端面材質、密封層次、活動量、泛水、排水、冷凝水、防潮、斷熱、最終屋頂標高及可檢修性。第一階段不以單層矽利康、開放縫或其他未討論細節代替決策。
