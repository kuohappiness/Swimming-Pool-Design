## 3. 已被取代的說法

| 原說法 | 取代原因 | 現行說法 |
| --- | --- | --- |
| 綠色區域是既有廁所與更衣室 | 衛星圖只能確認既有廁所基地，不能證明更衣室現況 | 綠色區域記錄為既有廁所基地，並納入重建規劃 |
| 入口與樓梯方案已確認 | 使用者明確表示精確位置稍後討論 | OPEN-001、OPEN-002 |
| 男女更衣室樓層已確認 | 樓層、大小及進出方式尚待整體比較 | OPEN-003 |
| 正式丈量與 CAD 是開始建模的必要前置 | 使用者無法取得，且本案是個人概念規劃 | 先採明確規劃值，未來參數化修訂 |
| V03 屋頂方向是當前答案 | V03 的坡向與文字紀錄存在衝突 | OPEN-004 |
| 每個規劃尺寸都要標「暫估」 | 造成畫面噪音，且不符合本案用途 | 數值直接顯示；依據與修訂狀態留在資料層 |

## 4. 風險與控制

| 風險 | 可能結果 | 控制方式 |
| --- | --- | --- |
| 衛星圖不是測量成果 | 基地尺寸或角度與現況不同 | 保存原圖、校準參數與修訂記錄；幾何全部參數化 |
| 手繪透視被誤當按比例圖 | 產生不合理尺寸或樓層 | 只提取空間意圖；數值由 DESIGN_BASIS 管理 |
| 待決事項被程式預設值掩蓋 | Viewer 看似完整但不可追溯 | schema 支援 deferred／缺值狀態，驗證禁止未標示的臨時值 |
| Viewer 與 DXF 漂移 | 不知道哪個成果可信 | 同一模型資料、同版號、同一組生成與測試流程 |
| 玻璃屋頂越界覆蓋服務空間 | 違反已確認設計 | 幾何測試檢查 roof zone 只與 pool hall 相交 |
| 公開衛星圖來源被忽略 | 缺少來源脈絡 | 保留畫面標示並在來源登錄表記錄 Google 及資料提供者 |

## 5. 下一個決策順序

建議依序處理：

1. 入口的主要到達方向與位置範圍。
2. 更衣室採一樓、二樓或分層方案。
3. 樓梯位置與乾濕動線。
4. 服務量體外輪廓。
5. 屋頂高低側方位。

這個順序能先解開最多下游幾何依賴，避免先畫出之後必須整體推翻的 3D 模型。
