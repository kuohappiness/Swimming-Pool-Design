# 中師實小游泳池改建｜工作流程與版本管理

## 1. 專案階段

### 階段 A：需求與概念整理

- 蒐集基地資料、使用需求、概念圖與現況資訊。
- 建立設計需求書、設計原則、Decision Log、Issue List 與 Risk List。
- 狀態：已完成第一輪整理，仍可持續更新。

### 階段 B：量體探索

- 建立基地比例與建築最大包絡。
- 優先配置最大泳池面積。
- 確認二樓量體、屋頂、入口、樓梯與景觀走道。
- 現階段不啟動 Design Freeze。

### 階段 C：平面配置

- 排入男、女各 20 間更衣／淋浴單元。
- 排入男廁、女廁、洗手台與機房。
- 檢討入口、樓梯、景觀走道及公共視線。

### 階段 D：立面與剖面

- 校正單斜玻璃屋頂位置與 10° 坡度。
- 設計長邊落地玻璃帷幕與結構分格。
- 檢討高低側淨高、二樓量體與泳池挑高。

### 階段 E：細部概念

- 門、欄杆、隔間、置物層板、掛勾、泳道線與救生設備。
- 材質、玻璃透明度、水面與簡易照明效果。

### 階段 F：概念定稿

- 在設計成熟後才進行 Concept Freeze。
- 定稿後再製作 SketchUp 匯入檔或 Ruby 建模腳本。
- 法規、結構、消防、無障礙、防水與機電另由專業人員深化。

## 2. 版本規則

- 主版本：V01、V02、V03……
- 必要時可使用小版本：V02a、V02b……
- 不直接覆蓋舊版本。
- 每版須包含變更摘要與待確認事項。

## 3. 每版同步成果

每次設計修改應同步更新：

1. 3D DXF。
2. 2D 一樓與二樓平面 DXF。
3. 立面與剖面 DXF。
4. HTML 互動式 3D 預覽。
5. 手機可看的靜態預覽圖。
6. 設計說明與變更紀錄。
7. Decision Log、Issue List、Risk List（必要時）。

## 4. 模型座標與單位

- 單位：毫米（mm）。
- X 軸：向東。
- Y 軸：向北。
- Z 軸：向上。
- 西側短邊為目前主入口基準。
- 所有輸出共用相同原點、座標與方位。

## 6. 品質檢查

### 幾何一致性

- 座向與衛星圖一致。
- DXF 與 HTML 使用相同尺寸。
- 平面、立面與剖面互相吻合。
- 屋頂方向與高低點一致。

### 空間需求

- 男、女各 20 間更衣／淋浴。
- 男廁 3 馬桶與 3 小便斗。
- 女廁暫定 5 間。
- 一樓設洗手台，二樓不設。
- 景觀走道與靠窗樓梯保留。

### 設計原則

- 泳池面積優先。
- 公共空間可看見泳池。
- 建築輕盈、透明、易管理。
- 未確認尺寸標示為暫估。
