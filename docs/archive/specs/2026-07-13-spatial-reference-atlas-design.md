# 空間參照圖集設計規格

- 日期：2026-07-13
- 類型：design
- 狀態：completed
- 任務：legacy
- 目標版本：0.1.0-atlas.1
- 完成日期：2026-07-13
- 適用專案：Swimming Pool Design

## 1. 目的

建立一套人與模型都能共同使用的空間參照系統，讓後續討論不再依賴「左邊、右邊、紅框附近」等會隨圖面方向改變的說法。

每個重要空間、入口、樓梯、立面及屋頂都有穩定 ID；平面圖、剖面圖、3D Viewer、DXF 與文字決策引用同一組名稱。設計比例修改時，ID 保持不變，幾何座標與尺寸則由單一模型更新。

## 2. 核准方案

採用「穩定編號＋建築格網＋樓層」方案，並與世界座標並存：

1. 人員討論層：樓層、建築格網、空間 ID 與圖號。
2. 模型運算層：真北向的世界座標與公尺數值。

不採只寫空間名稱的簡化方案，因為同名空間可能有多個位置；也不要求討論時每次使用完整 XYZ 數值，避免造成閱讀負擔與假精度。

## 3. 參照架構

### 3.1 世界座標

延續現行模型契約：

- `+X`：東
- `+Y`：北
- `+Z`：上
- 長度單位：公尺
- 角度單位：度

概念設計的本地原點定義為 `O-SITE-01`：主入口 `EN-01` 門檻中心在 L1 完成面的位置，座標為 `(0, 0, 0)`。未來若取得測量資料，只新增從本地座標到測量座標的 transform；既有 ID 與本地幾何關係不改名。

### 3.2 建築格網

格網用於圖面與日常對話，不取代世界座標：

- 字母軸 `A–F`：沿泳池長向，由泳池遠端／屋頂低側朝服務核心端／高側前進。
- 數字軸 `1–4`：由懸浮樓梯所在的長向玻璃牆朝對側長牆前進。
- 格網間距由模型幾何產生，不假設等距。
- 比例改變時可移動格線，但格線的相對順序與名稱保持穩定。

### 3.3 樓層

- `L1`：泳池大廳、入口、廁所及主要池畔層。
- `L2`：男女更衣與淋浴層。
- `RF`：屋頂層。

圖面標題、空間標籤及模型物件均明確帶有 level。跨樓層物件如樓梯可同時登錄 `fromLevel` 與 `toLevel`。

## 4. ID 規則

ID 使用大寫英文字母、數字及連字號；建立後不因名稱美化、尺寸或位置微調而重編。

| 類型 | 格式 | 範例 |
| --- | --- | --- |
| 原點／基準 | `O-<範圍>-NN` | `O-SITE-01` |
| 空間分區 | `Z-<功能>-NN` | `Z-PH-01` 泳池大廳、`Z-SC-01` 服務核心 |
| 入口 | `EN-NN` | `EN-01` 唯一日常人員入口 |
| 樓梯 | `ST-NN` | `ST-01` 懸浮樓梯 |
| 廁所 | `WC-<M/F>-NN` | `WC-M-01`、`WC-F-01` |
| 更衣淋浴整合單元 | `CS-<M/F>-NN` | `CS-M-01`、`CS-F-01` |
| 池體 | `PL-NN` | `PL-01` 主泳池 |
| 立面／牆面 | `F-<類型>-NN` | `F-GL-01` 長向玻璃牆 |
| 屋頂 | `RF-<類型>-NN` | `RF-GL-01` 泳池玻璃屋頂 |
| 圖面 | `REF-NNN` | `REF-101` L1 平面參照圖 |

`M` 與 `F` 只表示男性區及女性區，不推導空間大小。第一版替 40 個整合單元建立穩定 ID：每側 01–15 為正式單元，16–20 為擴充位置；壁掛櫃及踏階不另建 ID。

## 5. 第一版圖集

| 圖號 | 名稱 | 必須顯示 |
| --- | --- | --- |
| `REF-001` | 基地與方位圖 | 衛星底圖、基地範圍、真北、比例、道路、`O-SITE-01`、`EN-01` |
| `REF-101` | L1 平面參照圖 | 格網、泳池大廳、池體、入口、廁所、服務核心、`ST-01` 下端 |
| `REF-201` | L2 平面參照圖 | 格網、男女嚴格分區、更衣與淋浴區、`ST-01` 上端 |
| `REF-301` | 屋頂參照圖 | 玻璃屋頂範圍、10° 坡度、高側、低側、服務空間非玻璃屋面 |
| `REF-401` | A–A 剖面參照圖 | L1、L2、泳池挑高、雙鋼梯梁懸浮樓梯與中間平台、屋頂坡向與標高關係 |
| `REF-501` | 3D 軸測參照圖 | 主要 zone、入口、樓梯、玻璃牆、玻璃屋頂與北向 |

圖面不需要在第一版達到施工圖細節。它們的任務是提供穩定空間語言、比例關係及可核對的設計狀態。

## 6. 圖面標示

每張圖固定顯示：

- 圖號與名稱
- revision
- modelVersion
- 樓層或視圖方向
- 真北箭頭；剖面及室內視圖則標示觀看方向
- 單位與主要尺寸
- 格網
- ID＋中文名稱
- 狀態圖例
- 「概念設計，非施工用途」

每個標籤的最低內容為 `ID｜中文名稱`。尺寸直接顯示數值；證據來源、依據與修訂狀態保存在模型 metadata，不在每個尺寸旁標示「約」。

## 7. 狀態語言

不能只依靠顏色，線型與文字也必須能辨識：

| 狀態 | 顯示方式 |
| --- | --- |
| confirmed | 實線、正常飽和度、標籤 `CONFIRMED` |
| working | 琥珀色虛線、標籤 `WORKING` |
| deferred | 灰紫色點線或半透明區域、標籤 `OPEN-xxx` |
| legacy | 淡灰色、標籤 `LEGACY`，預設不顯示 |

男女區域可使用不同填色協助辨識，但分區邊界、ID 與文字必須獨立成立，不能讓顏色成為唯一資訊。

## 8. 資料所有權

空間參照資料是單一模型的一部分，不建立另一份手動座標真相。`model/project-model.json` 加入以下結構：

```json
{
  "referenceSystem": {
    "originId": "O-SITE-01",
    "worldAxes": { "x": "east", "y": "north", "z": "up" },
    "grid": {},
    "levels": [],
    "entities": []
  }
}
```

每個 entity 至少包含：

- `id`
- `type`
- `name`
- `level` 或 `fromLevel`／`toLevel`
- `gridLocation`
- `geometryRef`
- `status`
- `sourceIds`
- `revision`

原始照片、衛星截圖與手繪標註仍以 `SRC-*` 登錄，不直接改造成設計圖。參照圖由模型與來源關係生成，並清楚區分「觀察證據」與「設計詮釋」。

## 9. 產出形式

第一版以可縮放的 SVG／HTML 產生，理由如下：

- 手機與桌面皆可清楚縮放。
- ID、圖層與狀態可直接顯示或隱藏。
- 同一份幾何資料可重用於 Viewer 與 DXF。
- 可列印或另行輸出 PDF，而不讓 PDF 成為新的資料真相。

參照圖集的穩定入口定為 `reference/index.html`。正式 3D Viewer 完成後，在 Viewer 中加入相同 ID 的標籤圖層；點選 ID 可定位相同模型物件。第一版不實作複雜搜尋、註解協作或線上編輯器。

## 10. 討論範例

- 「把 `ST-01` 的下端維持在 `L1/B1`，上端朝服務核心接到 `L2/A1`。」
- 「`RF-GL-01` 只覆蓋 `Z-PH-01`，不得覆蓋 `WC-M-01` 或 `WC-F-01`。」
- 「`Z-CS-M-01` 與 `Z-CS-F-01` 在 `REF-201` 必須完全分離，各有 15 間正式單元及 5 間擴充位置。」
- 「請從 `REF-501` 顯示 `F-GL-01` 與 `ST-01` 的通透關係。」

實際格網位置由首版平面比例確定；上述範例說明語法，不替尚未生成的幾何宣告座標。

## 11. 驗證與完成條件

第一版參照圖集必須滿足：

1. 所有 ID 唯一且符合格式。
2. 每個圖面引用的 ID 都存在於模型 entity registry。
3. L1、L2、RF 的格網名稱與位置一致。
4. 真北、世界座標與 Viewer／DXF 一致。
5. 圖面 revision 與 modelVersion 可見。
6. confirmed、working、deferred、legacy 不只靠顏色區分。
7. 原始來源圖未被覆寫或重新壓縮。
8. 手機可閱讀標籤並縮放，不需依賴桌面 hover。
9. 同一物件在平面、剖面與 3D 圖使用相同 ID。
10. 不把 deferred 幾何畫成已確認空間。

## 12. 實作順序

1. 建立 referenceSystem schema 與 ID validator。
2. 建立 origin、levels、grid 與 entity registry。
3. 先產生 `REF-001`、`REF-101`、`REF-201`。
4. 空間比例確認後產生 `REF-301`、`REF-401`。
5. 與 3D 幾何共用資料後產生 `REF-501`。
6. 加入手機 smoke test、跨圖 ID 一致性檢查及靜態 build。

參照圖集先於完整 3D Viewer 建立，作為後續設計討論與模型驗證的共同索引。
