# 冬夏日照互動展示設計

- 日期：2026-07-14
- 類型：design
- 狀態：completed
- 任務：legacy
- 目標版本：0.1.0-atlas.3
- 完成日期：2026-07-14

## 1. 目的

把已確認的「平面＋剖面」互動提案正式放入 repo，讓非專業觀看者一眼理解：1F 不動，2F 水平旋轉控制反射方位，面池端牆外傾控制反射向下角度；目標是冬季上午利用日照、夏季不增加池面反射。

## 2. 範圍與非目標

正式頁面提供冬至／夏至、08:00～12:00、2F 水平旋轉與牆面外傾四項操作。它顯示太陽高度角、方位角、鏡牆法線、反射向下角及命中／避開判讀。

本頁不是建築定案、結構支點設計、眩光分析、材料反射率模擬、遮蔭研究、熱負荷或水溫預測。`+4.5°` 與 `9.5°` 只是能解釋設計意圖的操作預設。

## 3. 權威資料與計算

- 校名、經緯度、時區、建築長軸、模型版本從 `model/project-model.json` 取得。
- 泳池方向由建築本地 +X 方位反向推導，不在頁面硬編碼第二個答案。
- 太陽位置依 NOAA 公開近似式，由日期、時間、經緯度及 UTC offset 即時計算。
- 鏡面反射使用三維向量反射；平面方位偏差與剖面向下角必須同時通過才顯示命中。
- 操作門檻屬展示判讀，不寫回正式 L2 幾何。

## 4. 畫面架構

1. 頁首：基地經緯度、時區、長軸與泳池方向。
2. 左圖平面：固定 1F、旋轉 2F、面池端鏡牆、冬夏太陽方位。
3. 右圖剖面：鏡牆外傾、入射光、反射光與結果。
4. 控制：季節、時間、水平旋轉、外傾角。
5. 冬夏角度表：由同一計算函式產生。
6. 說明與限制：固定／可變分工、操作預設、`OPEN-011`。
7. 與空間參照圖集互相連結。

## 5. 行動裝置與可及性

920 px 以下改為單欄，320 px 仍不得水平裁切控制與文字。原生 `select`／`range` 保留鍵盤操作；兩張 SVG 具有 `<title>`／`<desc>`，結果區使用 `aria-live`。

## 6. 驗證

- 單元測試校驗已知冬至／夏至太陽角度與三維反射。
- 預設值必須得到冬至 09:00 命中、夏至 09:00 避開。
- validator 檢查校址、時區與來源。
- Vite 多頁 build 必須產生圖集與日照研究兩個入口。
- 人工檢查桌面及 390 px 手機寬度的版面及四項控制。
