## 正式 enhanced rendering

- `/3d-viewer/` 預設建立 enhanced runtime；`?rendering=baseline` 保留為明確診斷入口，必要材質建立失敗會自動回復 baseline 3D runtime，WebGL 不可用時仍保留靜態五場景內容。
- 清水模、玻璃、鋼、池磚、池畔、欄杆、鏡面、光電與水面使用 semantic PBR registry；8 個本地材質資產及程序 environment 皆有來源、作者、CC0-1.0、SHA-256、byte size、tier 與 required／optional manifest。
- 本地 PMREM environment、ACES／sRGB、per-scene exposure 與 stable PCF shadow 提供日光、柔光、雨天與反射；正式 high 使用直接渲染，半解析度 SSAO pipeline 保留為可選能力。
- surface seams／frames／coping、程序化植栽、人物尺度與設備全部標記 `visualOnly`／`collisionExcluded`，不改 canonical bounds、selectables、openings、collision 或 Walkthrough state。
- enhanced water 只讀與 `POOL-01` simulation 共用的水面標高、movement mode 與 cutaway state；low 關閉昂貴 normal／environment，水域與水下辨識仍保留。

## 畫質與相容性

- hardware desktop／mobile runtime 目標仍為平均 50／30 FPS；software renderer 直接從 low 啟動，硬體窄螢幕從 medium、桌機從 high 啟動，再由兩個持續超標觀測窗單向 high→medium→low。
- 1440 × 900 SwiftShader high／medium／low 的總 transfer 均為 214,109 bytes，低於 24／12／6 MiB；平均 FPS 為 1.68／1.76／14.89，p95 為 610.5／587.5／73.2 ms。
- 390 × 844 adaptive 在同一 software renderer 為 4.04 FPS、p95 260.9 ms；所有 deterministic 結果均高於同機 v0.6.7 baseline 的 80%。software FPS 只作相對防退化證據，不冒充硬體驗收。
- low 將 draw calls 由 1,678 降至 638、triangles 由 59,416 降至 27,118、shader programs 由 38 降至 22，且不改 scene、camera、layers、selection、player、collision 或 movement mode。
- 真實 `WEBGL_lose_context` restore 保留 scene、cutaway、selection 與 layers；dependency-injected optional environment failure 維持 enhanced，required material failure 回復 baseline，兩者都不產生 page error。
- TASK-064 首輪顯示半解析度 SSAO 在 1440 × 900 SwiftShader 只有 1.35 FPS，因此正式 high 取消全畫面 SSAO。這是依量測做的 release 優化，不降低 PBR、PMREM、ACES 或 PCF shadow。
