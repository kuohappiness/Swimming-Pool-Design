## 根因與修正

- 使用者的判斷正確：L2 服務中心 Y0 的灰色面不是玻璃反射，而是一片實際的清水模牆。
- canonical model 已定義男女更衣淋浴區分界 `l2.splitAxisY=8`，但 `scripts/viewer-data.mjs` 的 L2 手動白名單漏傳此欄位，使 9.0 × 3.3 m 男女分隔牆在 Viewer 中以未定義座標建立於 Y0、直接遮住玻璃。
- Viewer L2 payload 改為完整繼承 active L2 欄位，再疊加衍生值，避免日後新增 canonical 欄位時再次被白名單漏掉。
- model adapter 對缺失、非有限值、與男女區共用邊界不一致，或與 Y0 玻璃／樓梯區重疊的 `splitAxisY` 直接 fail closed，不再允許未定義值進入 Three.js。
- 男女更衣實體分隔牆固定並命名於 Y8；Y2.5 的 X32～X41 樓梯／更衣室連續清水模分隔牆維持不變。

## 撤回錯誤症狀修補

- 撤回 0.6.6 為了讓灰牆「看似玻璃」而加重的共用玻璃顏色、不透明度、clearcoat、反射與 emissive 設定，恢復一般淡藍透明玻璃基準。
- 保留 `DEC-114` 已確認的無黃色選取外接框，以及 SITE-XY Y0／Y14 固定視角。
- 修正巢狀構件選取 owner 被外層群組覆寫的問題；L2 Y0 玻璃現在會保持 `F-L2-Y0-01` 身分。
