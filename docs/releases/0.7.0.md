## 第一人稱漫遊

- `/3d-viewer/` 保留原 Inspect Viewer，另提供可逆的第一人稱模式；退出會恢復相機、場景、圖層、選取、固定視角／泳池剖視、panel scroll 與焦點。
- 桌機提供 WASD／方向鍵、Pointer Lock 或 drag-look fallback、Shift、Space、C／Ctrl 與 Esc；手機提供左右雙觸控面及 44 CSS px 上浮、下潛、返回與退出控制。
- 六個區域跳轉只使用由 active semantic entities 衍生、且通過 clear＋supported 驗證的 safe spawn：入口、L1 池畔、L2、L3、L3 露台與屋頂檢查區。

## 碰撞、樓梯與恢復

- 固定 1/120 秒步進共用 1.65 m 人眼與 0.28 m capsule；render FPS 或 quality 不改 simulation state。
- collision world 從 active walk surfaces、pool shell、openings、ST-01／ST-02 及 live `siteRoot.matrixWorld` 建立，不另存第二份 canonical geometry。
- `EN-01` 保持可通行；建築邊界、池壁、池底與樓梯側會阻擋 capsule。ST-01／ST-02 以連續坡面支援雙向 traversal。
- NaN、掉出範圍、unresolved penetration 與使用者返回只會回到已驗證 safe spawn，並清除速度。

## 水面與水下

- 可見水面、WaterVolume 與池底碰撞共用 `POOL-01` 衍生狀態；水面為池畔下 0.08 m，X3 淺端 1.20 m、X28 深端 1.50 m。
- walking／falling／surface／underwater transition 使用 hysteresis；支援浮力、三軸游泳、上浮、下潛、池壁／池底、clearance assisted climb 與返回池畔。
- 水下霧化、色調與 audio hook 可依 low quality／reduced motion 降級；音訊、pointer capture 或 pointer lock 失敗不阻斷核心移動。

## 效能與擴充邊界

- frame-time monitor 記錄平均與 p95；只有兩個持續超標觀測窗才會在本 session 單向 high→medium→low，避免 shader warm-up 誤降級與反覆震盪。
- 降級只影響 pixel ratio、shadow、水下效果與 camera motion，不改 player、collision、area 或 movement mode。
- `InputAdapter`、`MovementStrategy`、`VisualAssetAdapter`、`AreaRegistry` 與 `EnvironmentEffect` 是可替換窄介面；0.7.0 正式預設仍使用 baseline rendering。

## 版本與模型隔離

- `model/project-model.json` 只變更 package／model／active revision release metadata 與對應 owner ID；建築 bounds、標高、樓梯、池體、L3、鏡牆與 integration approvals 均未改動。
- V067 四張 current 圖面仍是未變的建築幾何基線；0.7.0 沒有冒充新建築圖面版本。
- solar inputs 與 registry `inputHash=3f9ae69b4dcde4e2dc88894f6e441b878a8b242a938f4f568419275282ddb999` 未變，狀態維持 `current`，沒有執行日照最佳化。

## 專業驗證界線

Walkthrough collision、樓梯坡面與游泳是概念 Viewer simulation，不代表建築法規、結構、消防、避難、無障礙、水處理、池體安全或施工核定。`integrationReview.professionalApprovals` 保持全數 `false`。
