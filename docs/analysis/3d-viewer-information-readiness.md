# 3D Viewer 資訊完整度與同步準備度

本文件是 3D Viewer 建模準備度、公開理念內容同步風險與後續補強順序的分析 owner。公開敘事以[理念文字正本](../public/swimming-pool-renovation-design-concept.md)為準；有效數值、決策與未決事項仍由 03～05 及個別分析 owner 管理。

## 1. 結論

目前資料**足以建立概念設計級 3D Viewer MVP**，可準確表達主要量體、空間關係、屋頂坡向、2F 旋轉、鏡牆外傾、樓梯、雨簾及新舊建築關係；但尚不足以宣稱為施工級 BIM、精密光熱模型或法規檢討模型。

Viewer 第一版必須把資訊分為三類：

- `confirmed`：依現行決策正常呈現。
- `working`：可以建模，但在資訊面板與圖層說明中標示為工作值。
- `deferred`：不得虛構施工細節；採省略、半透明概念界面或明確待決標示。

## 3. 目前會阻止「一次修改、全部同步」的問題

### 3.1 同一設計值仍有多個儲存位置

L2／屋頂高端／樓梯總升高的 4.500 m 目前分別存在於：

- `referenceSystem.levels[L2].elevation`
- `geometry.roof.highElevation.value`
- `geometry.stair.totalRise`

Validator 會檢查它們仍是 4.5，但尚未由單一 canonical elevation 推導其餘值。修改任一欄位不會自動遷移另外兩處。

### 3.2 Renderer 與文件仍含顯示用硬編碼

`reference/src/sheets.ts` 仍直接保存 `+4.500`、`4.5°`、`+2.910` 等文字；公開理念 Markdown 也直接保存 9.5°、8.5°、4.5°與 1.2 m。這些內容目前不會在模型修改後自動重寫。

### 3.3 Validator 鎖定舊答案，而不是鎖定關係

現行 validator 與測試會要求屋頂高端及樓梯總升高精確等於 4.5。這對防止 0.3.x 回歸有效，但不支援正式設計變更。0.4.0 必須把部分「等於常數」改為「符合核准決策與跨欄位不變條件」，同時保留歷史版本測試。

### 3.4 3D 必要工作幾何尚未正式入模

現有剖面使用 3.6 m 作為 L2 視覺量體高度，日照能量分析使用 3.0 m 鏡面工作高；兩者用途不同，均未成為完整 3D schema。正式 Viewer 前至少要補上：

- L2 視覺量體高度，標示為 working。
- Viewer 使用的 2F 旋轉支點，標示為 working 並連結 `OPEN-011`。
- 鏡牆完整牆面與分析有效鏡面區的區分。
- 概念材質 palette 與非施工承諾說明。

## 4. 0.4.0 模型與同步補強順序

1. 以 L2 elevation 作為樓層高度 canonical input；屋頂高端與樓梯總升高由關係推導。
2. `roof.lowElevation`、遠端牆標高及雨簾滴水端只作 derived output，不再保存可漂移的第二答案。
3. 將顯示文字改為 formatter 由模型值產生，不在 renderer 保存數字句子。
4. 為公開 Markdown 建立模型 token 解析器；對外顯示值由模型注入。
5. 補齊 3D Viewer 所需 working geometry，並保留 `OPEN-011`、`OPEN-013`、`OPEN-014`。
6. 以模型雜湊與內容雜湊標記 Viewer、擷取畫面、理念 HTML、PDF及 DOCX 版本。
7. 幾何變更時使相依的日照、眩光與熱分析進入 stale 狀態，不以文字替換冒充重新驗證。

## 5. 第一版 Viewer 可安全表達的範圍

- 簡化基地、真北與建築工作方位。
- 1F 建築殼體、泳池、池畔、長向玻璃牆與服務核心。
- L2 原核心、擴建量體、男女分區及工作旋轉支點。
- 4.5°玻璃屋頂、1.2 m 外挑、被動雨簾與概念回收水路。
- 面池端鏡牆的 +8.5°外傾與局部鏡面語彙。
- 懸浮樓梯的兩跑、平台、雙箱型梯梁、開放梯下及概念弦幕。
- 「向光、向雨、向人、向時間」四組固定敘事場景。
- confirmed／working／deferred 圖層與說明。

第一版不得加入虛構的機電設備尺寸、正式鏡面分格、結構節點、季節控制表皮、精密地形或未核准室內隔間。

## 6. 判定

資訊完整度已通過 **3D Viewer MVP 開始門檻**，條件是先完成模型單一來源補強、working/deferred 表達及自動同步檢查。施工級精度、最終材料與全年專業性能驗證仍不在 0.4.0 MVP 的完成定義內。
